Output Transformation

                            

                            
—

Converts printf calls to cout << chains. Format specifiers like %d, %s, %c, %f are stripped from the format string, and arguments are streamed via <<.

Converts scanf calls to cin >> chains. The format string is dropped entirely, and the address-of operators (&) are removed from variable references.

C → C++ Source Transpiler

                    
A mini compiler project that translates C source code into equivalent C++ programs using a rule-based token transformation approach.

                


                

                    
Compiler Pipeline

                    

                        

                            
1

                            

                                
Lexical Analysis

                                
Lexer.h / Lexer.cpp

                                
Reads the input C source and breaks it into meaningful tokens: Keywords, Identifiers, String Literals, Numbers, Symbols, Preprocessor directives, Whitespace, and Comments.

                            

                        

                        
→

                        

                            
2

                            

                                
Transformation

                                
Transformer.h / Transformer.cpp

                                
Applies rule-based conversions: printf→cout, scanf→cin, malloc→new, free→delete, struct→class, and header substitution.

                            

                        

                        
→

                        

                            
3

                            

                                
Code Generation

                                
CodeGenerator.h / CodeGenerator.cpp

                                
Reconstructs the modified token stream back into continuous, formatted C++ source code, preserving original whitespace and comments.

                            

                        

                    

                


                

                    
Known Limitations

                    
    
                        
  • Naive Parsing — Operates at the token level without an AST, so complex recursive structures may not parse perfectly.
    • 
                        
  • Printf Specifiers — Complex format strings like %04d or %.2f are stripped rather than mapped to <iomanip> equivalents.
    • 
                        
  • Malloc Typing — Uses new char[size] naively without interpreting the target pointer's type.
    • 
                        
  • No Semantic Analysis — Variable scoping, name shadowing, and type inference are not handled.
    •
